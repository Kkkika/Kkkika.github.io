我该怎么使用这些功能

英文内容 中文翻译 功能解释
Automatic 模块 自动模块 -
Split timers for automatic commit and sync 拆分自动提交与同步的计时器 启用后可分别为 “提交” 和 “同步” 设置不同的时间间隔
Auto commit-and-sync interval (minutes) 自动提交与同步间隔（分钟） 设置每 X 分钟自动提交并同步更改；设为 0（默认）则禁用
Auto commit-and-sync after stopping file edits 停止文件编辑后自动提交与同步 需提交间隔不为 0；开启后，停止编辑 1 分钟后自动提交（避免编辑时触发）；关闭后与编辑操作无关
Auto commit-and-sync after latest commit 最新提交后自动提交与同步 开启后，将上次自动提交时间戳设为最新提交的时间戳，减少手动提交时自动提交的频率
Auto push interval (minutes) 自动推送间隔（分钟） 设置每 X 分钟自动推送提交；设为 0（默认）则禁用
Auto pull interval (minutes) 自动拉取间隔（分钟） 设置每 X 分钟自动拉取更改；设为 0（默认）则禁用
Auto commit-and-sync only staged files 仅对暂存文件自动提交与同步 开启后仅提交暂存文件；关闭后提交所有已更改文件
Specify custom commit message on auto commit-and-sync 在自动提交与同步时指定自定义提交信息 开启后会弹出弹窗，让你填写自定义提交信息
Commit message on auto commit-and-sync 自动提交与同步时的提交信息 设置自动提交的信息模板，支持 1764051712000（日期）等占位符
Commit message 模块 提交信息模块 -
Commit message on manual commit 手动提交时的提交信息 设置手动提交的信息模板，支持 （更改文件数）等占位符
Commit message script 提交信息脚本 通过 “sh -c” 运行的脚本，可借助 AI 生成提交信息，支持 等占位符
1764051712000 placeholder format 1764051712000 占位符格式 自定义日期显示格式（如 YYYY-MM-DD HH:mm），更多格式参考 Moment.js
placeholder replacement 占位符替换内容 为设备设置自定义主机名，用于提交信息的占位符替换
Preview commit message 提交信息预览 点击 “Preview” 按钮可查看提交信息的最终效果
List filenames affected by commit in the commit body 在提交正文中列出本次提交影响的文件名 开启后，提交信息正文中会显示本次提交涉及的文件名
Pull 模块 拉取模块 -
Merge strategy 合并策略 设置将远程分支提交整合到本地分支的方式（当前选项：Merge）
Pull on startup 启动时拉取 开启后，Obsidian 启动时自动拉取远程仓库的提交
Commit-and-sync 模块 提交并同步模块 默认流程：暂存所有内容→提交→拉取→推送，用于保持本地与远程仓库同步
Push on commit-and-sync 提交并同步时推送 开启后，提交并同步时执行推送；关闭后仅执行提交和拉取
Pull on commit-and-sync 提交并同步时拉取 开启后，提交并同步时执行拉取；关闭后仅执行提交和推送
Line author information 模块 行作者信息模块 -
Show commit authoring information next to each line 在每行旁显示提交作者信息 可单独切换显示作者 / 日期；关闭后仅显示带颜色的侧边栏标识
History view 模块 历史视图模块 -
Show Author 显示作者 设置是否在历史视图中显示提交的作者（当前选项：Hide）
Show Date 显示日期 开启后，在历史视图中显示提交日期（使用 1764051712000 格式）
Source control view 模块 源代码控制视图模块 -
Automatically refresh source control view on file changes 文件更改时自动刷新源代码控制视图 文件变化时自动刷新视图，同时添加 “暂存” 等文件操作
Show branch status bar 显示分支状态栏 开启后显示分支状态栏，修改后需重启 Obsidian 生效
Show the count of modified files in the status bar 在状态栏显示已修改文件的数量 开启后，状态栏显示当前未提交的已修改文件数
Commit author 模块 提交作者模块 -
Author name for commit 提交的作者名称 设置提交时显示的作者姓名
Author email for commit 提交的作者邮箱 设置提交时关联的作者邮箱
Advanced 模块 高级模块 通常无需修改，特殊配置场景下调整
Update submodules 更新子模块 开启后，“提交并同步”/“拉取” 会处理子模块（需为子模块设跟踪分支）
Custom Git binary path 自定义 Git 可执行文件路径 设置 Git 程序的自定义路径（当前默认：git）
Additional environment variables 额外的环境变量 按 “KEY=VALUE” 格式添加环境变量，每行一个
Additional PATH environment variable paths 额外的 PATH 环境变量路径 添加额外路径到 PATH，每行一个
Reload with new environment variables 使用新环境变量重新加载 应用新环境变量；移除变量需重启 Obsidian
Custom base path (Git repository path) 自定义基础路径（Git 仓库路径） 设置 Git 仓库相对于库根目录的路径（Windows 用 “\” 代替 “/”）
Custom Git directory path (instead of ‘.git’) 自定义 Git 目录路径（替代 “.git”） 设置自定义 Git 目录路径，修改后需重启 Obsidian
Disable on this device 在此设备上禁用 开启后，当前设备上禁用该插件（此设置不同步）
Support 模块 支持模块 -
Donate 捐赠 可捐赠支持插件的持续开发
Debugging and logging 调试与日志 通过 “Ctrl+Shift+I” 打开控制台查看插件日志

令牌权限对比

GitHub

Personal access tokens（个人访问令牌）

Tokens classic（经典令牌）

Generate new token classic（生成经典新令牌）

Full control of private repositories（对私人仓库的完全控制）

  • repo: 访问私有仓库的权限。

  • workflow: 访问 GitHub Actions 工作流的权限。

  • admin:org: 管理组织的权限（仅限组织所有者）。

  • delete_repo: 删除仓库的权限。

Gitee

  1. 日常代码操作（拉取 / 推送项目）
    • 必选：projects（查看、创建、更新项目，满足代码推拉需求）
    • 可选：pull_requests（如果需要提交 / 管理 PR）
  2. 管理项目问题 / 代码片段
    • 若要处理 Issues：加选issues
    • 若要管理代码片段：加选gists

不建议勾选的权限（非必要情况）

  • keys（公钥管理）、hook（Webhook 管理）

  • groups（组织管理）、enterprises（企业管理）

  • emails（邮箱信息查看）

    这些权限涉及账号敏感操作，非必需时不要开放。

  • 这是一个相邻兄弟选择器（+），选中两个连续的 .cm-line 元素
  • 前半部分：.cm-line:not(:has(> br:only-child)) - 前一个行

代码片段 含义 作用
.cm-line 类选择器 选中所有Obsidian编辑器行
:not(...) 否定伪类 排除符合括号内条件的元素
:has(...) 关系伪类 必须包含符合括号内条件的子元素
> 子选择器 直接子元素（不是后代）
br:only-child br标签且是唯一子元素 该行只有一个<br>元素
+ 相邻兄弟选择器 选择紧跟在后的兄弟元素
